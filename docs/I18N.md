# Block 6.4.1 — Internationalization

## Strategy

SequrAI uses a **cookie-based locale** (`sequrai_locale`) without URL prefixes.

This avoids breaking:

- GitHub OAuth callbacks
- Webhooks
- MCP endpoints
- Deep links to scans and reports
- API routes

Locale resolution order:

1. `profiles.locale` (authenticated users)
2. `sequrai_locale` cookie
3. `Accept-Language` header (first visit only)
4. Fallback: `en`

## Architecture

- `lib/i18n/` — core utilities (detect, translate, server, client provider)
- `messages/{en,es}/*.json` — namespaced copy
- `components/shared/LanguageSelector.tsx` — accessible language switcher
- `components/shared/I18nShell.tsx` — server wrapper for `I18nProvider`

No third-party i18n library was added. The stack uses typed JSON namespaces, `Intl` for dates/numbers, and a small `createTranslator()` with plural support.
